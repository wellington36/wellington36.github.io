Introdução

JavaScript é uma linguagem de programação orientada a objetos. É uma linguagem pequena e leve. Dentro de um ambiente host, o JavaScript pode ser conectado a objetos de seu entorno para fornecer controle sobre eles.

O JavaScript contém uma biblioteca padrão de objetos, como Array, Data e Math, e um conjunto básico de elementos de linguagem, como operadores, estruturas de controle e instruções. O JavaScript pode ser usado para uma variedade objetivos, por exemplo:

  • Client-side JavaScript fornece os meios para controlar um navegador e seu Document Object Model.
  • Server-side JavaScript fornece os meios para executar o JavaScript em um servidor.
JavaScript e Java

JavaScript e Java são semelhantes em alguns aspectos, mas fundamentalmente diferentes em outros. A linguagem JavaScript se assemelha a Java, mas não possui a tipagem estática e a verificação de tipo forte de Java. O JavaScript segue a maioria das sintaxes de expressão Java, convenções de nomenclatura e construções básicas de fluxo de controle, razão pela qual foi renomeado de LiveScript para JavaScript.

Em contraste com o sistema de classes de tempo de compilação do Java construído por declarações, o JavaScript suporta um sistema de tempo de execução baseado em um pequeno número de tipos de dados que representam valores numéricos, booleanos e string. JavaScript tem um modelo de objeto baseado em protótipo em vez do modelo de objeto baseado em classe mais comum. O modelo baseado em protótipo fornece herança dinâmica; ou seja, o que é herdado pode variar para objetos individuais. JavaScript também suporta funções sem quaisquer requisitos declarativos especiais. As funções podem ser propriedades de objetos, executando como métodos de tipagem flexível.

JavaScript é uma linguagem de forma muito livre em comparação com Java. Você não precisa declarar todas as variáveis, classes e métodos. Você não precisa se preocupar se os métodos são públicos, privados ou protegidos e não precisa implementar interfaces: Variáveis, parâmetros e tipos de retorno de função não são explicitamente tipados.

Declaração de variaveis
Você pode declarar uma variavel de 3 formas:

Com o comando var. Por exemplo, var x = 42; Está sintaxe pode ser usada para variaveis locais e globais.

Simplesmente atribuindo o valor. Por exemplo, x = 42; Está é sempre uma variavel global. O que pode gerar programas perigosos.

Ou com o comando let. Por exemplo, let x = 42; Está forma pode ser usada para variaveis de scopo de bloco.

Variaveis globais

As variáveis globais são, na verdade, propriedades do objeto global. Em páginas da web, o objeto global é window, então você pode definir e acessar variáveis globais usando a sintaxe window.variable.

Logo, você pode acessar variáveis globais declaradas em uma janela ou quadro de outra janela ou quadro especificados.

Tipos de dados

O padrão ECMAScript mais recente define sete tipos de dados:

  • Seis tipos de dados primitivos:

    • Bollean. true ou false.
    • null. Denota o valor vazio.
    • undefined. Uma propriedades de nivel alto que denota um valor não definido.
    • Number. um número, 42 ou 3.14159, por exemplo.
    • String. um texto, "Hello", por exemplo.
    • Symbol. Um tipo de dados cujas instâncias são únicos e imutáveis.
  • e Objetos
Embora esses tipos de dados sejam uma quantidade relativamente pequena, eles permitem que você execute funções úteis com seus aplicativos. Objetos e funções são os outros elementos fundamentais da linguagem. Você pode pensar em objetos como contêineres nomeados para valores e funções como procedimentos que seu aplicativo pode executar.
Declaração if...else
Usamos a declaração if para executar condicionais, e o opcional else para caso a condição do if não ocorra, por exemplo: if (condicional) { codigo_1; } else { codigo_2; } condicionais pode ser qualquer expressão que resulte num true ou false. Se a condição por verdadeira executa o codigo_1 se não executa o codigo_2. Você também pode computar multiplas condições, fazendo: if (condicional_1) { codigo_1; } else if (condicional_2) { codigo_2; } else if (condicional_n) { codigo_n; } else { codigo_last; } No caso de várias condições, apenas a primeira condição lógica avaliada como verdadeira será executada. Para executar várias instruções, agrupe-as em uma instrução de bloco ({ ... }) . Em geral, é uma boa prática sempre usar instruções de bloco, especialmente ao alinhar instruções if: if (condicional) { codigo_1_roda_se_condicional_ŕ_true; codigo_2_roda_se_condicional_é_true; } else { codigo_3_roda_se_condicional_é_false; codigo_4_roda_se_condicional_é_false; } É aconselhável não usar atribuições simples em uma expressão condicional, porque a atribuição pode ser confundida com igualdade ao examinar o código. Por exemplo, não use o seguinte código: if (x = y) { /* codigo aqui */ } Se você precisar usar alguma atribuição em uma expressão condicional, uma prática comum é colocar parênteses adicionais ao redor da atribuição. Por exemplo: if ((x = y)) { /* codigo aqui */ }
